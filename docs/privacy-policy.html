ClawChat Privacy Policy

Last updated: July 14, 2026

Summary: ClawChat is a local-first Android chat and agent app. Your conversations, settings, tools, skills, diagnostics, and Alpine workspace are stored on your device unless you choose an action that sends content to a model provider, a web target, or an optional user-authorized OpenClaw remote agent.

1. Scope

ClawChat is an open-source Android application maintained by the ClawChat project. It embeds an Alpine Linux workspace for user-initiated tool execution and connects to AI model providers using credentials and endpoints that you configure.

This policy describes the ClawChat application. It does not replace the privacy terms of the model providers, websites, remote agents, or operating-system services you choose to use through the app.

2. Data stored on your device

ClawChat stores app data locally, including:

  • Conversation history, drafts, attachments, exported files, and local recovery records.
  • Model/provider profiles, custom base URLs, model selections, and app preferences.
  • API keys and other secrets saved through Android secure storage or the local Alpine workspace.
  • Installed skills, extension metadata, local memory, diagnostics exports, and update/rollback records.
  • The Alpine root filesystem and files created by tools inside the app workspace.

ClawChat does not include analytics, advertising SDKs, or a project-operated telemetry service.

3. When data can leave the device

ClawChat can send data outside the device only through features you configure or invoke:

  • Direct model providers: prompts, selected conversation context, tool schemas, and relevant outputs may be sent to providers such as Anthropic, OpenAI-compatible endpoints, DeepSeek, OpenRouter, xAI, or other endpoints you configure.
  • Optional OpenClaw remote agent: when you explicitly authorize and enable it for a conversation, conversation text may be sent to that external agent. The app shows the connector name instead of exposing credentials.
  • Web and network tools: user-initiated tools such as Web Fetch, Web Search, skill import, update checks, and file downloads contact the target services needed for that action.
  • System sharing and browser actions: if you use Android share sheets, external browser links, phone, SMS, calendar, contacts, or install intents, Android and the selected target app handle the data you pass to them.

Provider and remote-agent requests are governed by the service you choose. Review those services before sending sensitive content.

4. Permissions

ClawChat requests Android permissions to support local workspace and agent features:

  • INTERNET and network state: model requests, configured endpoints, skill/update downloads, and web tools.
  • Foreground service, notifications, wake lock, and boot: keep user-initiated setup, terminal, cleanup, and agent tasks visible and recoverable.
  • Display over other apps (SYSTEM_ALERT_WINDOW): support optional floating or overlay-style visibility for user-started local work. Android sends you to system overlay settings, where you decide whether to grant or revoke this access.
  • Ignore battery optimizations (REQUEST_IGNORE_BATTERY_OPTIMIZATIONS): help keep user-started local workspace, terminal, setup, cleanup, or foreground agent tasks from being stopped unexpectedly. ClawChat hands you to Android battery optimization settings; the app does not grant this permission by itself.
  • Manage external storage and file providers: import, export, attachment, workspace, and update handoff flows.
  • Microphone and speech services: voice input and transcription when you start voice features.
  • Calendar, contacts, call, and SMS: phone-integration tools. Higher-risk call and SMS actions remain gated by app settings and Android permissions.
  • Request package installs: hand off a verified update package to the Android system installer after explicit user action.

5. Diagnostics, privacy mode, and secrets

Diagnostics and run traces are local developer/support tools. They are not uploaded automatically. When you export diagnostics, ClawChat attempts to redact secrets and avoid exposing raw prompts, tool payloads, endpoint identifiers, or credential values. You should still review exported files before sharing them.

Privacy mode masks detected environment-variable values before shell output is sent to a model. It does not change what a third-party provider may receive when you intentionally send prompts or context to that provider.

6. Updates and extensions

ClawChat supports signed update and extension workflows with local preview, explicit consent, and rollback records. The app does not silently install updates or extensions. Skill imports and update checks may contact the URLs or release sources involved in the action.
